# Networking Wiki

A knowledge base for networking concepts, protocols, architectures, operations, and troubleshooting notes.

The goal is to keep practical networking knowledge easy to find, connect, and expand over time.

## Structure

```text
docs/
├── concepts/
├── protocols/
├── architectures/
├── operations/
└── troubleshooting/
```

## Sections

### Concepts

Mechanisms and mental models that apply across networking systems.

Examples:

- Packet flow
- Encapsulation
- MTU
- Control plane and data plane
- Convergence
- Failure domains

### Protocols

Protocol behavior, packet exchanges, timers, states, and protocol-specific details.

Examples:

- ARP
- ICMP
- TCP
- DNS
- DHCP
- OSPF
- BGP
- STP
- HTTP CONNECT

### Architectures

Design patterns, topology choices, traffic-flow patterns, and component placement.

Examples:

- Forward proxy
- Reverse proxy
- NAT designs
- Internet edge
- Campus network
- Spine-leaf
- Segmentation
- CDN and origin shielding

### Operations

Repeatable procedures used to run, maintain, or validate networks.

Examples:

- Pre-change checks
- Post-change validation
- Interface turn-up
- Route validation
- Configuration backup
- Certificate rotation

### Troubleshooting

Symptom-first investigation paths and debugging notes.

Examples:

- No connectivity
- Packet loss
- High latency
- DNS resolution failure
- DHCP failure
- MTU blackhole
- Proxy connectivity failure
